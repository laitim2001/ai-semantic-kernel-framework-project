## 🔴 MUST (強制要求)
1. 所有 AI/LLM 呼叫**必須使用真實 Azure OpenAI**，絕不可 mock/simulation
2. 執行前**必須**確認環境健康：`curl http://localhost:8000/health`
3. **必須**完整執行測試，不可跳過任何步驟
4. **必須**記錄所有 LLM 實際響應內容（完整，非摘要）
5. **必須**提供包含「說明 + 實際執行內容」的整合報告

## 🟡 SHOULD (應該做)
1. 執行前檢查 Docker 服務：`docker-compose ps`
2. 記錄每個步驟的輸入/輸出/耗時
3. 發現問題時記錄完整錯誤訊息
4. 將測試結果與預期結果比對
5. 對相關功能進行影響範圍評估

## 🟢 MAY (可以做)
1. 參考歷史測試結果進行趨勢比對
2. 對測試結果提供改進建議
3. 識別潛在的邊界條件或風險

## ⛔ MUST NOT (禁止)
1. **禁止**使用 mock、simulation 或假資料
2. **禁止**跳過測試步驟或階段
3. **禁止**只提供說明而省略實際執行內容
4. **禁止**在失敗時自行修改測試標準或預期
5. **禁止**省略或摘要 LLM 響應（需完整記錄）
6. **禁止**未經確認環境就開始執行測試
