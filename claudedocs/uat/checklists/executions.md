## 測試步驟詳情

### 1.1 執行列表顯示

```yaml
前置條件:
  - 系統有至少 1 個執行記錄

測試步驟:
  1. 導航到 /executions 頁面
  2. 確認列表正確載入
  3. 確認顯示: 執行 ID、Workflow 名稱、狀態、開始時間

預期結果:
  - 列表按時間倒序顯示
  - 狀態顏色正確 (running=藍, completed=綠, failed=紅)
  - 無數據時顯示空狀態
```

### 2.1 啟動新執行

```yaml
前置條件:
  - 有一個可執行的 Workflow

測試步驟:
  1. 導航到 Workflow 詳情頁
  2. 點擊 "Execute" 或 "Run" 按鈕
  3. (可選) 填寫輸入參數
  4. 確認執行

預期結果:
  - 創建新的執行記錄
  - 狀態變為 "running"
  - 可在執行列表中看到新執行
```

### 2.2 取消執行

```yaml
前置條件:
  - 有一個正在執行中的任務

測試步驟:
  1. 找到 running 狀態的執行
  2. 點擊取消按鈕
  3. 確認取消

預期結果:
  - 執行狀態變為 "cancelled"
  - 相關資源被釋放
```

### 3.1 執行詳情頁面

```yaml
測試步驟:
  1. 從列表點擊執行記錄
  2. 查看詳情頁面

預期結果:
  - 顯示完整執行信息
  - 顯示每個節點的執行狀態
  - 顯示輸入/輸出數據
  - 顯示執行時間
```
