# 🎯 情況2: 開發任務準備 - 開始新功能開發前的準備

> **使用時機**: 收到開發任務後，開始編碼前的準備工作
> **目標**: 讓 AI 助手充分理解任務需求和相關代碼
> **適用場景**: Sprint 任務開始、新功能開發、需求分析

---

## 📋 Prompt 模板 (給開發人員)

```markdown
你好！我需要開始一個新的開發任務。

**任務**: [描述任務名稱和簡要說明]

請幫我：

1. 理解任務需求
   - 任務的目標是什麼？
   - 需要交付什麼？
   - 有什麼驗收標準？

2. 分析相關代碼
   - 這個任務涉及哪些模組？
   - 查看相關的現有代碼結構
   - 識別需要修改的文件

3. 規劃實作步驟
   - 列出具體的實作步驟
   - 估計每個步驟的工作量
   - 識別潛在的風險或阻礙

4. 準備開發環境
   - 確認分支策略
   - 檢查相關依賴
   - 準備測試環境

請用中文回答，並給出清晰的執行計劃。
```

---

## 🤖 AI 助手執行步驟

### Step 1: 理解任務需求 (2 分鐘)

```bash
# 1. 如有相關文檔，閱讀任務說明
Read: docs/03-implementation/sprint-execution/sprint-XX/story-XX.md (如有)

# 2. 理解任務背景
分析用戶描述的任務目標和預期結果
```

### Step 2: 分析相關代碼 (3 分鐘)

```bash
# 1. 定位相關模組
Glob: backend/src/api/v1/{相關模組}/
Glob: backend/src/domain/{相關模組}/

# 2. 閱讀核心代碼
Read: 相關的 service.py, routes.py, models.py

# 3. 檢查現有模式
Grep: "相關關鍵字" 搜尋現有實作模式
```

### Step 3: 規劃實作步驟 (2 分鐘)

```markdown
# 📝 任務分解

## 任務概述
- **名稱**: [任務名稱]
- **目標**: [交付目標]
- **影響範圍**: [涉及模組]

## 實作步驟
| 步驟 | 描述 | 預估時間 | 檔案 |
|------|------|----------|------|
| 1 | ... | Xmin | path/to/file.py |
| 2 | ... | Xmin | path/to/file.py |

## 風險識別
- [潛在風險 1]
- [潛在風險 2]

## 驗收標準
- [ ] 功能完成
- [ ] 測試通過
- [ ] 代碼審查
```

### Step 4: 準備開發環境 (1 分鐘)

```bash
# 1. 確認分支
Bash: git status
Bash: git checkout -b feature/任務名稱

# 2. 確認服務運行
Bash: docker-compose ps
Bash: curl http://localhost:8000/health
```

---

## ✅ 驗收標準

AI 助手應該完成以下準備工作：

1. **任務理解**
   - 清楚了解任務目標和交付物
   - 識別了相關的模組和文件

2. **代碼分析**
   - 閱讀了相關的現有代碼
   - 理解了現有的設計模式

3. **實作計劃**
   - 提供了明確的步驟分解
   - 估計了每個步驟的工作量

4. **環境準備**
   - 確認了開發分支
   - 確認了服務運行狀態

5. **風險識別**
   - 識別了潛在的技術風險
   - 提出了應對策略
