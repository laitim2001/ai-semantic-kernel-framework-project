### Design decision (model-invoked lazy-load; system-bundled markdown; ride the persona seam for the prompt block; the `read_skill` tool for the load; no FE / no wire / no DB)

- **US-1 is "the discovery substrate"**: a NEW `agent_harness/skills/` module (Cat 5 Prompt Construction ŌĆö skills are a prompt-augmentation/progressive-disclosure capability). `Skill` is a frozen dataclass (`name` kebab-id, `description` one-line, `instructions` full body). `SkillRegistry` mirrors `ToolRegistryImpl` (name-keyed dict + `register` / `get` / `list`) plus a `from_dir(path)` classmethod that reads `*.md`, splits the YAML frontmatter (`---` ŌĆ” `---`) via the existing `yaml.safe_load` (the `capability_matrix.from_yaml` precedent ŌĆö no new dependency), and takes the post-frontmatter body as `instructions`. A module-level singleton `get_default_skill_registry()` lazy-loads `backend/config/skills/` on first call (the bundled-config-at-startup pattern). Two REAL bundled skills ship: `code-review.md` (a structured-review skill: Summary / a Risks table with severity / concrete fixes) + `summarize.md` (condense a thread/document into Decisions / Action Items / Open Questions) ŌĆö both with instruction bodies a real LLM visibly follows (the drive-through proves the output shape changes with vs without the skill).
- **US-2 is "discover ŌåÆ load ŌåÆ follow, wired into the main flow"**: (i) **the cheap discovery layer** ŌĆö `render_catalog_block(skills) -> str` produces a compact "## Available Skills" block (`- {name}: {description}` lines + one instruction: "Call `read_skill(name)` to load a skill's full instructions before applying it"); `build_handler` appends it to the resolved `system_prompt` (the persona seam) when the registry is non-empty. (ii) **the expensive load layer** ŌĆö a `read_skill` ToolSpec (Cat 2; `input_schema {name: str}`, `risk_level=LOW`, `hitl_policy=AUTO` ŌĆö read-only) + `make_read_skill_handler(registry)` (returns the named skill's `instructions` framed as "# Skill: {name}\n\n{body}\n\nFollow these instructions for the current task.", or a clear "unknown skill" error); `make_default_executor` gains a `skill_registry` opt-in that registers the spec + handler (mirroring the `echo` registration + the `teammate_mailbox` opt-in precedent) so `read_skill` reaches the LLM's tool list. (iii) **the wiring** ŌĆö `build_handler` gains a `skill_registry` param: it appends the catalog block to `system_prompt` AND passes the registry to `make_default_executor`; the chat `router` passes `skill_registry=get_default_skill_registry()`. The result: the LLM sees the available skills cheaply, self-selects one when relevant, calls `read_skill`, receives the full instructions, and follows them ŌĆö the CC lazy-load loop, validated in the õĖ╗µĄüķćÅ (ń┤äµØ¤ 2).
- **System-bundled, not per-tenant**: the bundled skills are system-global (no `tenant_id`, no DB) ŌĆö exactly like `tools_registry` / `capability_matrix.yaml`. The "## Available Skills" block is byte-identical for every tenant (cache-stable for prompt caching; no per-tenant cache fragmentation; no tenant data in skills ŌåÆ no isolation surface). Per-tenant custom skills (DB-backed) are a deferred slice (`AD-Skills-Per-Tenant-Catalog`).
- **No new wire event**: `read_skill` is an ordinary tool ŌĆö its call + result stream through the existing Cat 2 tool-call SSE rendering (the Inspector tool view shows `read_skill(name)` + the returned instructions). Wire count stays **24**, zero codegen diff. A dedicated "skill loaded" Inspector affordance is deferred (`AD-Skills-Inspector-Affordance`).
- **Rejected**: system auto-match injection (Option B ŌĆö a pre-loop matcher injects the best skill; less CC-faithful + less agentic ŌĆö the model doesn't self-select); user `/skill` slash-command (Option C ŌĆö defers the model-invoked core + needs FE slash plumbing ŌĆö a later epic slice); per-tenant DB-backed skills this spike (premature ŌĆö YAGNI, system-bundled proves the loop first); a new SSE event for skill loads (the tool stream already carries it ŌĆö no codegen risk for a thin spike); rewiring `make_chat_prompt_builder` / `DefaultPromptBuilder.__init__` / `_build_system_section` to thread a catalog (a wider blast radius than appending to the already-proven `system_prompt` seam ŌĆö deferred unless Day-0 disproves the seam); placeholder/stub skills (AP-4 ŌĆö the bundled skills must be real + drivable).

### 3.1 SkillRegistry + loader + bundled skills (US-1)
