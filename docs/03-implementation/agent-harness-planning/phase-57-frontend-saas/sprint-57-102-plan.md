## 0. Background

The harness-deepening proposal's 収斂洞察 2: TEAMMATE multi-turn ("parent mid-task messages child") and the chat live-injection gap are the **same loop primitive** — B1 (Sprint 57.101) built the `MessageInbox` ABC + the `_run_turns` drain seam; B2 is the second payoff (a child loop's inbox backed by a parent-reachable channel). B2a wires the TEAMMATE child onto that primitive; B2b adds the live UI producer.

**The load-bearing Day-0 finding (the proposal's own deferred constraint, confirmed by reading code, not speculation)**: the parent BLOCKS on the child. `dispatcher.spawn()` allocates the `subagent_id`, emits `SubagentSpawned`, then `asyncio.create_task(_track_and_emit())` (`dispatcher.py:262`); `task_spawn`'s handler then `await dispatcher.wait_for(subagent_id)` (`tools.py:116`) which `await`s the task (`dispatcher.py:284`). So the parent turn is suspended until the teammate finishes. "The parent reasons mid-child-run and injects a new instruction" therefore requires non-blocking spawn + teammate lifecycle management — which the proposal §2.3 explicitly defers ("detached 長壽 teammate（需生命週期管理）後置"). The honest v1 keeps await-completion and demonstrates the child inbox with an **independent** writer (the chat user) — and because that writer is a **separate HTTP request**, the backing must be the module-level B1 `InjectionRegistry` (keyed by the child's `subagent_id`), NOT the per-request `MailboxStore` (which cannot bridge requests — the same reason B1 needed a module registry). That endpoint + its FE are **B2b**. B2a wires + unit-proves the child inbox; it does not claim a UI-drivable inbox.

### 3.0 Architecture (TEAMMATE becomes FORK-with-a-mailbox-and-an-inbox; loop.py untouched; await-completion preserved)

```
PARENT chat run (open SSE stream)
  parent LLM → tool_call task_spawn(mode="teammate", task)
    dispatcher.spawn(TEAMMATE) → subagent_id = uuid4(); emit SubagentSpawned
    _teammate.execute(subagent_id, parent_session_id, task, budget):
        inbox = inbox_factory(subagent_id)            # QueueMessageInbox over InjectionRegistry[tenant][subagent_id]
        child = teammate_child_loop_factory(budget, inbox)   # recursion-safe subset + send_to_parent + message_inbox=inbox

            emitter(SubagentChildEvent(subagent_id, inner=ev))   # 57.96 relay → Inspector Tree (incl. send_to_parent ToolCall)
            track last LLMResponded.content + LoopCompleted.total_tokens
            #  ↑ a child turn boundary drains `inbox` via the B1 seam (loop.py:2026) — [] in B2a prod (no producer yet)
        reports = mailbox.drain(parent_session_id, "parent")   # the child's send_to_parent messages (await-completion: all at once)
        summary = fold(reports, final_answer)                  # parent reads via task_spawn summary
        return SubagentResult(success, summary, tokens_used)
    task_spawn handler: result = await dispatcher.wait_for(subagent_id)   # BLOCKS (await-completion, unchanged)
        → returns {summary, …} → parent LLM integrates → parent's final answer

B2b (NOT this sprint): POST /chat/{id}/subagents/{subagent_id}/inject → InjectionRegistry[tenant][subagent_id].put → the child's
  next turn drains it (the seam already in place) → FE inject-to-teammate control + render.
```
