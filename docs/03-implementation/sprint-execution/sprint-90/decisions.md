# Sprint 90 技術決策記錄

## 決策列表

### D90-1: mem0 版本選擇

**決策**: 使用 `mem0ai>=0.0.1`

**理由**:
- 使用最小版本要求，確保相容性
- 允許自動升級到更新的修復版本
- 符合 Python 依賴管理最佳實踐

---

### D90-2: Qdrant 存儲模式

**決策**: 使用本地檔案系統存儲

**選項**:
1. 本地檔案系統 (選擇)
2. Qdrant Cloud 服務
3. 自託管 Qdrant 容器

**選擇**: 選項 1

**理由**:
- 簡化開發環境設置
- 無需額外服務依賴
- mem0 內建支持本地 Qdrant
- 對於開發和小規模部署足夠

---

### D90-3: 測試 Mock 策略

**決策**: 使用 pytest-mock 和 unittest.mock

**理由**:
- 隔離外部 API 調用 (OpenAI, Anthropic)
- 避免在測試中產生費用
- 確保測試的可重複性
