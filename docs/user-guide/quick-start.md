## 首次登錄

### 步驟 1: 訪問平台

打開瀏覽器，訪問 IPA 平台 URL：
```
https://ipa-platform.azurewebsites.net
```

### 步驟 2: 登錄

1. 輸入您的電子郵件地址
2. 輸入密碼
3. 點擊「登錄」按鈕

> **提示**: 如果您忘記密碼，點擊「忘記密碼」鏈接進行重置。

### 步驟 3: 設置個人資料（可選）

首次登錄後，建議完善您的個人資料：
1. 點擊右上角的用戶圖標
2. 選擇「設置」
3. 填寫顯示名稱和通知偏好

## 創建第一個 Agent

Agent 是執行任務的 AI 助手。讓我們創建一個簡單的 Agent：

### 步驟 1: 進入 Agent 管理

1. 點擊側邊欄的「Agents」
2. 點擊「創建 Agent」按鈕

### 步驟 2: 填寫 Agent 信息

```yaml
名稱: 客服助手
描述: 處理客戶查詢和問題
指令: |
  你是一個專業的客服助手。
  請友善、專業地回應客戶問題。
  如果無法解決，請轉交給人工客服。
模型: gpt-4
```

### 步驟 3: 配置工具（可選）

如果需要，可以為 Agent 啟用工具：
- 網頁搜索
- 計算器
- 外部 API 調用

### 步驟 4: 測試 Agent

1. 點擊「測試」按鈕
2. 輸入測試消息：「我想查詢訂單狀態」
3. 查看 Agent 回應
4. 滿意後點擊「保存」

## 創建第一個工作流

工作流定義了任務的執行順序。讓我們創建一個簡單的工作流：

### 步驟 1: 進入工作流管理

1. 點擊側邊欄的「Workflows」
2. 點擊「創建工作流」按鈕

### 步驟 2: 基本信息

```yaml
名稱: 客戶服務流程
描述: 處理客戶服務請求的自動化流程
類別: 客戶服務
```

### 步驟 3: 設計流程

一個基本的工作流包含：

```
開始 → Agent 處理 → 結束
```

進階流程可以包含：

```
開始 → Agent 分析 → 審批節點 → Agent 回覆 → 結束
```

### 步驟 4: 保存工作流

1. 檢查流程設計
2. 點擊「保存」按鈕
3. 工作流將顯示在列表中

## 執行工作流

### 手動執行

1. 在工作流列表中找到您的工作流
2. 點擊「執行」按鈕
3. 輸入執行參數（如有需要）
4. 點擊「開始執行」

### 查看執行狀態

執行開始後：
1. 自動跳轉到執行詳情頁面
2. 實時查看執行進度
3. 如果需要審批，您會收到通知

### 執行狀態說明

| 狀態 | 說明 |
|------|------|
| 🔵 運行中 | 工作流正在執行 |
| 🟡 待審批 | 等待人工審批 |
| 🟢 已完成 | 執行成功完成 |
| 🔴 失敗 | 執行過程中發生錯誤 |
