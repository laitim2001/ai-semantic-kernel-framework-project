## 系統概述

IPA Platform 是一個企業級 AI Agent 編排平台，讓您能夠：

- **創建智能工作流**: 使用拖放式編輯器設計自動化流程
- **整合多個系統**: 連接 ServiceNow、Dynamics 365、Teams 等企業應用
- **AI 驅動決策**: 利用 Azure OpenAI 實現智能判斷
- **人機協作**: 在關鍵步驟加入人工審批檢查點

### 核心概念

| 概念 | 說明 |
|------|------|
| **工作流 (Workflow)** | 一系列按順序執行的自動化步驟 |
| **節點 (Node)** | 工作流中的單一操作單元 |
| **Agent** | AI 驅動的智能助手，執行特定任務 |
| **檢查點 (Checkpoint)** | 需要人工審批的暫停點 |
| **執行 (Execution)** | 工作流的一次運行實例 |

## 登入系統

### 步驟 1: 訪問平台

在瀏覽器中輸入平台 URL：
```
https://your-company.ipa-platform.com
```

### 步驟 2: 輸入認證資訊

1. 輸入您的 **電子郵件**
2. 輸入您的 **密碼**
3. 點擊 **登入** 按鈕

> **提示**: 首次登入時，請使用管理員提供的臨時密碼，並立即更改。

### 步驟 3: 多因素認證 (可選)

如果啟用了 MFA，請：
1. 打開您的認證應用程式 (如 Microsoft Authenticator)
2. 輸入 6 位數驗證碼
3. 點擊 **驗證**

## 創建第一個工作流

讓我們創建一個簡單的 "Hello World" 工作流：

### 步驟 1: 進入工作流頁面

1. 點擊導航欄的 **工作流**
2. 點擊右上角的 **+ 新增工作流**

### 步驟 2: 設定基本資訊

```yaml
名稱: My First Workflow
描述: 我的第一個自動化工作流
觸發方式: 手動
```

### 步驟 3: 添加節點

1. 從左側工具欄拖拽 **開始節點** 到畫布
2. 添加一個 **日誌節點**，設定訊息為 "Hello, IPA Platform!"
3. 添加一個 **結束節點**
4. 連接各節點

### 步驟 4: 保存工作流

1. 點擊 **保存** 按鈕
2. 系統會驗證工作流配置
3. 看到 "保存成功" 提示

> **提示**: 工作流默認為「草稿」狀態，需要啟用後才能執行。

### 步驟 5: 啟用工作流

1. 點擊 **啟用** 按鈕
2. 確認啟用
3. 狀態變為「活躍」

## 執行工作流

### 手動執行

1. 在工作流列表中找到您的工作流
2. 點擊 **執行** 按鈕 (▶)
3. 可選：輸入執行參數
4. 點擊 **確認執行**

### 查看執行狀態

執行開始後：

1. 系統自動跳轉到執行詳情頁
2. 實時查看每個節點的執行狀態
3. 綠色 ✅ = 完成，黃色 ⏳ = 進行中，紅色 ❌ = 失敗

### 執行狀態說明

| 狀態 | 說明 |
|------|------|
| `PENDING` | 等待執行 |
| `RUNNING` | 正在執行 |
| `WAITING_APPROVAL` | 等待人工審批 |
| `COMPLETED` | 執行成功 |
| `FAILED` | 執行失敗 |
| `CANCELLED` | 已取消 |

### 常見問題

**Q: 如何重新執行失敗的工作流？**
A: 在執行詳情頁點擊「重試」按鈕。

**Q: 工作流可以定時執行嗎？**
A: 可以！在創建工作流時選擇「排程」觸發方式，設定 Cron 表達式。

**Q: 如何分享工作流給其他用戶？**
A: 在工作流設定中添加協作者，或將工作流設為團隊可見。
