  關鍵決策先行問題

  在挑槓桿組合前，建議先釐清：

  1. Subagent 模式 vs Agent Team 模式 — 判斷邏輯該相同還是不同？
    - Subagent：每個 agent 是一次性 worker，同質性高 → 可以按「子任務數」直接對應
    - Team：每個成員有固定角色 → 角色該不該上場是 binary（登場 / 不登場），而非「數量」
  2. 我們在優化什麼目標？
    - 最小化 LLM 成本（少 agent 優先）
    - 最大化答案品質（多角度優先）
    - 匹配用戶感受的「合理性」（避免一句話觸發 8 個 agent 的違和感）
  3. 判準該是規則性還是統計性？
    - 規則性：keyword / pattern / 任務類型 → 明確但難覆蓋邊界
    - 統計性：query 長度 / 領域數 / 動詞數 → 有彈性但可能黑盒

  建議起點

  先花 15-20 分鐘釐清問題 1 和 2（模式區分 + 優化目標），再回來評估槓桿組合。因為不同目標會導向截然不同的解法（例如若目標是「成本」，槓桿 C
   已足夠；若是「感受合理性」，需要 A+D 組合）。

  要先討論哪個問題？或有其他切入角度？
